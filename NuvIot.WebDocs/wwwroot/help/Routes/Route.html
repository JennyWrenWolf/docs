Route

Routes are used to create a path through Pipeline Execution Modules that will be used to process Incoming Messages from your devices.

A Device Configuration can have many routes and each route is associated with a Message Id.

Here is the basic flow.

  1. The Planner will determine the device configuration from the Device Id and the message from the Message Id from the incoming message.
  2. Based the type of message and device configuration, the planner will look up the corresponding route.
  3. The Planner will then create a set of instructions that reference pipeline modules that will executed to process the message.

Creating a Route

  1. Ensure you have created and configured the Pipleine Modules you will use in the route.
  2. Open a Device Configuration
  3. Click Add Route - A default route will be created where you can select pipeline modules.
  4. Select a Message Definition, doing so will associate this route with a message as it arrives from the device.
  5. Click on a node in our route and select a Pipeline Module.
  6. To add a Transmitter, click the Add Transmitter button above route canvas.
  7. Configure Mappings between applicable nodes.

  3. We Execute a workflow called Simple Workflow that you have created
  4. The Output Translator takes the Output Commands from the workflow and convert it to an outgoing messages that can be either sent back to the device or a 3rd party endpoint with a Transmitter.
  5. The Transmitter modules take those Outogoing Messages and either send them back to the device via the original listener, or forward to a 3rd party endpoint.

Mappings

To allow for maximum reusability, some pipeline modules require mappings.

  • Input Translator to Workflow Inputs Map the fields from messages that were parsed with the Input Translator to Inputs on the Workflow Module that will process the message.
  • Workflow Output Commands to Output Translator Map the outputs created by the Device Workflow to messages that will be created by the Output Translator.
  • Output Translator to Transmitter Map the messages you created with the Output Translator to the Transmitter that will send them.
