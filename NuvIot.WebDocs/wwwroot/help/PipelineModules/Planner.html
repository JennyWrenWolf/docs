Planner

The planner is responsible for determine how the incoming messages should be processed. Although you can implement your own custom planner, which you will use your custom message and device as configured device configuration to find a match, once a match is found, the route associated with that device configuration is determined and the steps to process the message are queued up for execution.

Unless you have some very specific needs you should be able to use the standard planner.
