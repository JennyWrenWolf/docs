Parsing Strategies

Values can be parsed to find a Message Id and Device ID or values from the message payload.

Device Id and Message Id

One of the responsibilities of the planner is to extact the device an message id. You can will need to create parsers to extact these values.

Payload

Your message will likey contain a number of attributes you want to extact to be used in workflow processing, these values defined as part of the message and considered th emessage payload.

Parsing Options

There are a number of different options for extracting values from your message

Headers - For HTTP/HTTPS message that includes a HTTP Header
Query String - Parse values from an HTTP(s) GET/POST/PUT or DELETE Query String
Path - Extract Values from the URL Path for an HTTP(s) request
Topic - Extract Values from a message that uses a topic such as a publish scribe protocol
Binary Messages - Extact values based on position and data types from a binary message
Delimited Messages - Use delimiters such as commas or tabs to extact values
JSON - Extract Message Values from a JSON document from any source
XML - Extract Message Values from an XML document from any source
String with Regular Expression - Build a Regular Expression to extract multiple values from a string based payload String with Position - Use start index and length to extact values from a string based message
