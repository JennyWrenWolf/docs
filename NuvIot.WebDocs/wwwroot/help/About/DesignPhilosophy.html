Design Philosophy

Our Design Philosopy is based on a pragmatic approach to building software that focus on quickly implementing requirements in a system that will stand the test of time, both in terms of scalability and flexibility.

Here a number of axioms that we adopted when designing NuvIot:

  1. Currently building real IoT applications is hard requiring very specialized talents
  2. Security needs to start at the infrastructure, it can't be added on at a later time
  3. Scalability can be achived by breaking the problem into seperate independent processing steps
  4. High Availability should be as simple as selecting a configuration
  5. There are a number of similar building blocks in every IoT application
  6. Each IoT application is a collection of building blocks or microservices
  7. With the right level of abstraction, similar buidling blocks can be tailored via configuration rather than coding
  8. The majority of an IoT application can be assembled by connecting configured building blocks or microservices
  9. It is impossible to 100% configure every requirement in a non-trivial system
  10. Integration Points should be provided to seemlessly add both scripts and custom code
  11. Adding custom code should let the developers focus only on their specific needs, not infrastructure
  12. Just because something isn't hard doesn't mean it's not industrial grade
