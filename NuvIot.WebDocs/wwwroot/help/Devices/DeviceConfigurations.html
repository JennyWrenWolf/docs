Device Configurations

A device configuration consists of two parts:

  • A collection of routes
  • A set of Properties

When you create a solution you specify a set of listeners and a planner as well as any supported device configurations. The listeners job is to listen for messages, once a message is received it is passed to the listener that determine the message id and device id. With those two pieces of information it will attempt to find a match for a device configuration based on known devices and supported messages types. If a match is found it will then use the steps as defined in the route to process the rest of the message.

Device Properties can be used as part of your scripts.
